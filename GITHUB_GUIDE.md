# GitHub プッシュ手順ガイド (VS Code)

このドキュメントでは、VS Code を使用してこのプロジェクトを GitHub にプッシュする手順をまとめます。

## 1. .gitignore の確認

現在のプロジェクトの `.gitignore` は適切に設定されており、以下のファイルは GitHub にプッシュされないようになっています：

- `node_modules/` (依存関係)
- `.next/`, `out/` (Next.js ビルド成果物)
- `build/`, `dist/` (ビルド出力)
- `.turbo` (Turborepo キャッシュ)
- `.env*.local` (機密情報を含む環境変数)
- `.vscode/` (エディタ設定)

**注意:** リモートに公開したくない `.env` ファイルなどは、必ず `.gitignore` に含まれていることを確認してください。

---

## 2. VS Code での手順

### ステップ 1: Git リポジトリの初期化 (未実施の場合)
1. 左側の **「ソース管理 (Source Control)」** アイコン (`Ctrl+Shift+G`) をクリック。
2. **「リポジトリを初期化する (Initialize Repository)」** をクリック。

### ステップ 2: 変更のステージングとコミット
1. 変更されたファイルの一覧にある **「+ (すべてをステージング)」** アイコンをクリック。
2. 上部のテキストボックスにコミットメッセージを入力（例: `feat: initial setup`）。
3. **「コミット (Commit)」** ボタンをクリック。

### ステップ 3: リモートリポジトリの追加
1. ブラウザで [GitHub](https://github.com/new) にアクセスし、新しいリポジトリを作成します。
2. 作成後の画面に表示される `https://github.com/ユーザー名/リポジトリ名.git` という URL をコピーします。
3. VS Code のソース管理パネルの **「… (三点リーダー)」** > **「リモート」** > **「リモートの追加...」** を選択。
4. コピーした URL を貼り付け、リモート名として `origin` を入力します。

### ステップ 4: プッシュ
1. ソース管理パネルの **「…」** > **「プッシュ (Push)」** を選択、または下部のステータスバーにある **「同期 (Sync Changes)」** をクリックします。
2. 初回プッシュ時は、GitHub への認証（サインイン）が求められる場合があります。

---

## 3. ターミナルでのコマンド実行 (代替手段)

VS Code の統合ターミナルで以下を順に実行することでも完了できます：

```bash
# 1. GitHub リモートの追加
git remote add origin <GitHubのURL>

# 2. ブランチ名を main に変更 (デフォルトが master の場合)
git branch -M main

# 3. 全ファイルをステージングしてコミット
git add .
git commit -m "Initial commit"

# 4. GitHub にプッシュ (-u で次回以降 git push だけで済むよう設定)
git push -u origin main
```
